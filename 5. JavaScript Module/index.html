JavaScript 模块化

山茶树和葡萄树 - xinlu.ink - 2019/05/23

面临场景

  • - 早期互联网环境下的应用场景简单，JavaScript 语言设计得较随(粗)意(糙)
  • - script 标签引/嵌入
  • - 时代演变，通过浏览器(内核)交互的应用越来越多越大
  • - Web 开发者编写的代码越来越多越复杂
  • - 语言的约束性宽松，人的约束性...
  • - 需要一套‘作用域隔离、组织构建(依赖、合并、压缩、混淆)’机制

我们经历了

  • - 所有的代码写入到单个 JavaScript、CSS 文件
  • - 手动地在各 HTML 文件中引入全部的 JavaScript、CSS 文件
  • - 变量暴露在全局环境，出现命名冲突
